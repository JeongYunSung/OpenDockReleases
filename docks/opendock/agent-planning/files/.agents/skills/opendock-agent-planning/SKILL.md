# Agent Planning

## 절차

1. `.opendock/docks/agent-planning/PLANNING.md`를 읽고 요청한 계획 작업을 바로 수행합니다.
2. 목표와 범위를 확인하고 실행 가능한 작은 단계로 나눕니다.
3. 의존성, 위험, blocker와 완료 근거를 필요한 수준으로 기록합니다.
4. 정형화된 문서가 필요할 때만 `TASK_PLAN.md`, `FINDINGS.md`, `PROGRESS.md` 중 필요한 템플릿과 섹션을 선택합니다.
5. 진행 중 확인된 사실과 상태를 구분하고 완료 항목에는 근거를 남깁니다.
6. 사용자가 검토를 요청하면 현재 결과물만 도메인 가이드 기준으로 직접 검토합니다.

계획은 작업을 관리하는 수단이며 승인된 범위를 넓히는 근거가 아닙니다.
