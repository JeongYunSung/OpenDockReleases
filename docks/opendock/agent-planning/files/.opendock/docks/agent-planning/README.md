# Agent Planning 사용 안내

복잡한 작업을 목표, 범위, 단계, 의존성, 진행 상태와 완료 근거로 나눕니다.

## 바로 사용하기

1. `.opendock/docks/agent-planning/PLANNING.md`에서 계획 기준을 확인합니다.
2. AI에게 “이 요구사항을 실행 가능한 단계와 완료 조건으로 나눠줘”처럼 결과를 직접 요청합니다.
3. 정형화된 문서가 필요할 때만 `TASK_PLAN.md`, `FINDINGS.md`, `PROGRESS.md` 중 필요한 템플릿과 섹션을 사용합니다.

## 검토 요청

사용자가 검토를 요청하면 AI가 현재 결과물만 `PLANNING.md` 기준으로 직접 살펴보고 목표와 범위, 단계 순서, 의존성, 완료 근거, 위험과 blocker를 설명합니다. 명시적 요청이 없으면 과거 계획이나 프로젝트 전체로 범위를 넓히지 않습니다.

사용자가 만든 결과물은 사용자 소유이며, Dock이 설치한 파일이 아니라면 OpenDock 업데이트나 제거가 삭제하지 않습니다.
