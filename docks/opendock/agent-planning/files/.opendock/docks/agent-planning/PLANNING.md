# 작업 계획 가이드

복잡한 작업을 실행 가능하고 추적 가능한 수준으로 나눕니다.

## 구성 후보

- Goal: 완료하려는 결과
- Scope: 포함하거나 제외할 범위
- Steps: 순서가 있는 실행 단위
- Dependencies: 선행 조건과 외부 의존성
- Evidence: 완료를 뒷받침할 테스트, 검토, 로그 또는 산출물
- Risks: 실패 가능성과 대응
- Handoff: 다음 작업자가 이어받을 내용

모든 항목을 채울 필요는 없습니다. 작업 규모와 위험에 필요한 항목만 선택합니다.

## 상태 예시

- `todo`: 아직 시작하지 않음
- `in_progress`: 작업 중
- `blocked`: 외부 입력이나 상태 변화 없이는 진행 불가
- `needs_review`: 검토 필요
- `done`: 완료 근거가 있음

상태 이름은 프로젝트 관례에 맞게 바꿀 수 있습니다. `done`에는 확인 가능한 근거를 남깁니다.

## AI 검토 기준

사용자가 검토를 요청하면 AI는 현재 결과물에서 목표와 범위의 명확성, 단계 순서, 의존성, 완료 근거, 위험, blocker와 다음 행동만 직접 확인합니다.
