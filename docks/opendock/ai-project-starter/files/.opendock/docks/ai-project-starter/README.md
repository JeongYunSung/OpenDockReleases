# AI Project Starter 사용 안내

프로젝트 근거를 바탕으로 AI 협업 맥락, 역할, 규칙, 작업 절차, 품질 기준과 onboarding을 `.ai/`에 정리합니다.

## 바로 사용하기

1. `.opendock/docks/ai-project-starter/AI_PROJECT_STARTER_PLAYBOOK.md`에서 구성 원칙을 확인합니다.
2. AI에게 “이 저장소 근거로 `.ai/PROJECT.md`의 목표, 규칙, 도구와 onboarding을 만들어줘”처럼 결과를 직접 요청합니다.
3. 별도 작업 메모가 유용할 때만 `RUN.md`를 열고 프로젝트에 필요한 섹션만 사용합니다.

## 검토 요청

사용자가 검토를 요청하면 AI가 현재 `.ai/` 결과물만 플레이북 기준으로 직접 살펴보고 프로젝트 근거, 역할과 승인 경계, 규칙, 도구, 작업 절차, 품질 기준, 보안과 onboarding의 공백을 설명합니다. 명시적 요청이 없으면 기존 vendor 설정이나 프로젝트 전체로 범위를 넓히지 않습니다.

사용자가 만든 결과물은 사용자 소유이며, Dock이 설치한 파일이 아니라면 OpenDock 업데이트나 제거가 삭제하지 않습니다.
