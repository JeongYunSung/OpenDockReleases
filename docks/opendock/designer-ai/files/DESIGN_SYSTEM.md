# Design System

## Spacing

- Define a base spacing scale.
- Use consistent section, component, and control spacing.
- Avoid one-off offsets unless the layout documents why.

## Typography

- Define display, heading, body, label, and code styles.
- Keep line length readable.
- Avoid viewport-scaled font sizes.

## Color

- Define semantic tokens for surface, text, border, action, success, warning,
  and danger.
- Check text contrast before shipping.

## Components

- Document expected states: default, hover, focus, disabled, loading, empty,
  error, and success.
- Prefer existing components over new variants.
