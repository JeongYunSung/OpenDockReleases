# Trip Planner 사용 안내

## 먼저 이렇게 요청하세요

- 알려줄 것: 여행 날짜, 인원, 예산, 선호, 보행·식사 같은 제약과 예약 상태
- 받을 결과: `trips/` 아래에 날짜별 일정, 이동 시간, 예산, 예약 확인, 우천 대안과 안전 계획
- 예시: "10월 교토 4박, 부모님과 하루 8천 보 이하 조건이야. 동선과 우천 대안을 trips/kyoto.md로 정리해줘."

## 사용 순서

1. 필요한 정보를 알려주고 빼야 할 개인정보도 함께 말합니다.
2. 교통·날씨·휴무·예약 정보를 확인하고 사실, 가정, 추천을 나눠 일정에 넣습니다.
3. 결과는 사용자 소유 `trips/` 아래에 저장합니다.
4. 작업 메모가 필요하면 선택 양식에서 관련 섹션만 골라 사용합니다.

평소에는 별도 준비 없이 바로 작업합니다. 사용자가 검토를 요청하면 AI가 현재 결과물만 `.opendock/docks/trip-planner/TRIP_PLANNING_PLAYBOOK.md` 기준으로 직접 검토합니다.

## 주의할 점

- 가격, 좌석, 객실과 영업 여부를 보장하지 않고 다시 확인할 시점을 적습니다.
- 여권 번호, 예약 코드, 상세 주소와 연락처는 일정에 넣지 않습니다.
- 바뀔 수 있는 정보는 링크와 확인 날짜를 남기고 실제 행동 전에 다시 확인합니다.
- `trips/` 아래의 결과는 사용자 소유이며 Dock 업데이트나 제거 대상이 아닙니다.
