# 여행 계획 플레이북

평소에는 별도 준비 없이 바로 계획을 만듭니다. 작업 메모가 필요하면 선택 템플릿에서 관련 섹션만 사용하고, 사용자가 검토를 요청하면 AI가 현재 결과물만 이 플레이북 기준으로 직접 검토합니다.

## 1. 입력 계약

필수 입력은 여행자 구성, 출발·귀국 날짜, 총예산과 통화, 선호, 이동·식이·접근성·숙박 제약입니다. 이름 대신 역할이나 가명을 사용하고 여권 번호, 예약 코드, 상세 주소와 개인 연락처는 수집하지 않습니다.

정보가 부족하면 임의로 채우지 말고 `미결정 사항`에 질문, 영향, 결정 기한을 기록합니다.

## 2. 조사와 근거

- 공식 교통기관, 기상기관, 시설 운영자, 정부 안전 안내를 우선합니다.
- 시간 민감 정보마다 URL, 접근일, 적용 날짜와 변동 가능성을 남깁니다.
- `사실`은 출처 또는 사용자 입력으로 확인된 내용, `가정`은 계획을 위해 임시로 둔 내용, `추천`은 근거와 상충관계를 포함한 선택 제안입니다.
- 가격은 통화, 포함·제외 항목, 확인 시점과 완충률을 표시합니다. 재고나 예약 가능성을 추정해 확정하지 않습니다.

## 3. 일정 설계

하루마다 시작·종료 시각, 장소, 체류 시간, 이동 수단과 이동 시간, 식사와 휴식을 적습니다. 지도상 거리만 보지 말고 환승, 대기, 짐 이동, 체크인, 보행량과 동행자 속도를 반영합니다.

핵심 일정에는 우천·휴무 대안을 연결하고, 변경 비용이 큰 예약부터 결정 시점을 표시합니다. 하루를 과밀하게 채우지 않고 회복 시간과 지연 완충을 둡니다.

## 4. 예산과 예약

교통, 숙박, 식비, 입장, 통신, 보험, 예비비를 분리합니다. 1인당·전체 금액과 환율 가정을 구분합니다. 예약 섹션에는 대상, 공식 링크, 예약 필요 이유, 취소 조건 확인 항목, 재확인 시점을 씁니다. 실제 구매나 예약은 사용자의 명시적 승인 없이 수행하지 않습니다.

## 5. 안전과 개인정보

공식 비상 연락 경로, 대사관·영사관 확인 방법, 보험과 약품 운반 확인 사항을 안내하되 전문 조언을 대신하지 않습니다. 공유본에서는 숙소 상세 주소, 티켓 바코드, 예약번호와 실시간 위치를 가립니다. 원본 메모도 목적에 필요한 기간만 보관합니다.

## 6. 완료 기준

- 요청에 필요한 일정, 이동, 예산, 예약, 대안과 안전 항목만 선택해 작성합니다.
- 출처와 접근일, 사실·가정·추천이 분리되어 있습니다.
- 이동 시간, 예산 내역, 예약 재확인, 짐, 우천 대안과 비상 계획이 서로 모순되지 않습니다.
- 미결정 사항이 질문과 영향으로 남아 있습니다.
- 검토 요청에서는 AI가 현재 결과의 최신성, 현실성, 접근성과 표현을 이 플레이북 기준으로 확인합니다.
