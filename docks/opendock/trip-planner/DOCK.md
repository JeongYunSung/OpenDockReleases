# Trip Planner

## 무엇을 알려주면 무엇이 나오나요

- 알려줄 것: 여행 날짜, 인원, 예산, 선호, 보행·식사 같은 제약과 예약 상태
- 정리되는 것: `trips/` 아래에 날짜별 일정, 이동 시간, 예산, 예약 확인, 우천 대안과 안전 계획
- 요청 예시: "10월 교토 4박, 부모님과 하루 8천 보 이하 조건이야. 동선과 우천 대안을 trips/kyoto.md로 정리해줘."

## 설치되는 안내

- `.opendock/docks/trip-planner/README.md`: 가장 쉬운 사용 순서
- `.opendock/docks/trip-planner/TRIP_PLANNING_PLAYBOOK.md`: 자세한 작성 기준
- `.opendock/templates/trip-planner/RUN.md`: 필요할 때 일부만 쓰는 선택 양식

## 사용 순서

1. 필요한 정보를 받고 민감한 값은 먼저 뺍니다.
2. 교통·날씨·휴무·예약 정보를 확인하고 사실, 가정, 추천을 나눠 일정에 넣습니다.
3. 결과를 사용자 소유 `trips/` 아래에 저장합니다.
4. 평소에는 별도 준비 없이 바로 작업하며, 양식은 필요한 섹션만 골라 사용합니다.

## 검토 요청

사용자가 검토를 요청하면 AI가 현재 결과물만 `TRIP_PLANNING_PLAYBOOK.md` 기준으로 직접 검토합니다.

## 안전

- 가격, 좌석, 객실과 영업 여부를 보장하지 않고 다시 확인할 시점을 적습니다.
- 여권 번호, 예약 코드, 상세 주소와 연락처는 일정에 넣지 않습니다.
- 웹의 가격·규정·운영 정보는 출처와 확인 날짜를 남기고 중요한 결정 전에 다시 확인합니다.
- 사용자가 만든 `trips/` 파일은 Dock을 업데이트하거나 제거해도 삭제하지 않습니다.
