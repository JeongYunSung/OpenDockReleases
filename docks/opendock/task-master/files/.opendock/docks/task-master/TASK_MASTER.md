# Task Master 사용 가이드

## 권장 흐름

1. 입력 PRD, issue, 요구사항의 범위와 성공 기준을 확인합니다.
2. 변경될 파일과 provider 설정을 확인한 뒤 승인된 명령만 실행합니다.
3. 생성된 task의 중복, 크기, dependency, acceptance criteria를 검토합니다.
4. 실제 project 구조와 우선순위를 반영해 task를 수정합니다.

## 좋은 task 기준

- 결과물과 완료 조건이 명확합니다.
- 검증 명령 또는 리뷰 기준이 있습니다.
- dependency, blocker, risk가 표시됩니다.

## 금지

- API key, credential, private PRD를 작업 기록에 저장하지 않습니다.
- 생성된 task를 검토 없이 확정하거나 자동 실행하지 않습니다.
