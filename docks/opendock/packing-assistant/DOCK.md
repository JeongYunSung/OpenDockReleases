# Packing Assistant

여행 조건을 과도하지 않은 수량 기반 짐 목록으로 바꾸는 도크입니다. 기후와 활동이 다른 여러 날의 여행을 준비하는 사람, 항공 수하물·숙소 세탁·의약품 운반 조건을 함께 확인해야 하는 사람, 체크 가능한 목록을 만드는 에이전트를 위한 품질 게이트입니다.

## 설치되는 내용

- 프로젝트용 `README.md`, `AGENTS.md`, `HARNESS.md` 관리 블록
- `PACKING_PLAYBOOK.md` 수량·재착용·세탁 플레이북
- `opendock-packing-assistant` skill과 품질 게이트 workflow
- `.opendock/templates/packing-assistant/RUN.md` 실행 기록 템플릿
- `.opendock/harness/opendock__packing-assistant/check.mjs` 로컬 하네스

## 프롬프트 예시

- "12월 삿포로 5박, 야외 산책과 온천 일정, 기내 7kg 제한에 맞춰 수량이 있는 체크리스트와 입기·세탁 계획을 만들어줘."
- "제주 자전거 여행의 최신 날씨와 숙소 세탁 가능 여부를 반영해 전자기기, 서류, 약품 주의와 출발 직전 목록을 정리해줘."
- "현재 packing 실행에서 현지 구매 가능한 항목과 수하물·의약품 caveat가 빠졌는지 검사해줘."

## 사용 흐름

1. 템플릿을 `.opendock/runs/packing-assistant/<run-id>/manifest.md`로 복사합니다.
2. 목적지, 날짜, 날씨 출처, 활동, 숙소, 교통, 수하물과 직접 제공된 의료 제약을 기록합니다.
3. 결과를 `packing/` 아래에 작성하고 수량·카테고리·재착용·세탁·현지 구매를 연결합니다.
4. 대상 파일을 선언하고 하네스를 실행해 실패를 수정합니다.
5. Codex가 여행 맥락과 누락 위험을 별도로 검토하며 실제 규정은 출발 전 다시 확인합니다.

## 안전과 한계

- 약을 처방하거나 용량·복용 중단·대체를 지시하지 않습니다. 기존 처방, 의료진·약사 안내와 목적지 공식 반입 규정을 확인하도록 안내합니다.
- 날씨, 항공사 수하물과 보안 규정은 변경될 수 있으므로 출처 URL, 접근일과 재확인 시점을 남깁니다.
- 여권 번호, 예약 코드, 처방전 세부 식별자와 건강 진단은 산출물에 넣지 않습니다. 필요한 문서는 항목명만 기록합니다.
- 하네스는 실제 가방 무게, 제품 적합성, 통관 승인 또는 외부 모델의 결정론을 보장하지 않습니다.

## 언어와 산출물 소유권

Run manifest의 `Language`는 `ko` 또는 `en`으로 기록하고 산출물도 같은 언어로 작성합니다. `packing/` 아래 결과는 사용자 소유이며 Dock manifest가 관리하지 않으므로 OpenDock update와 uninstall이 삭제하지 않습니다.
