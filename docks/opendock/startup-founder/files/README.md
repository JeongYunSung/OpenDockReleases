# Startup Founder Workspace

Business model, ICP, GTM, pricing, and founder strategy workspace.

## Who This Is For

Solo founders, startup CEOs, indie hackers, and very small teams.

## What This Dock Sets Up

AI-assisted strategy stays grounded in customer pain, willingness to pay, distribution, and experiments.

## Installed Agent Context

- `AGENTS.md`: shared instructions for Codex and other agent runtimes.
- `CLAUDE.md`: Claude Code project memory and skill routing notes.
- `GEMINI.md`: Gemini-compatible workspace context.
- `.agents/skills/opendock-startup-founder/SKILL.md`: OpenDock/OMA-style reusable skill instructions.
- `.codex/skills/opendock-startup-founder/SKILL.md`: Codex skill entry for this workspace role.
- `.claude/skills/opendock-startup-founder/SKILL.md`: Claude Code skill entry for this workspace role.
- `.cursor/rules/opendock-startup-founder.mdc`: Cursor project rule for this dock.

## Start Here

1. Define ICP and painful use case before product claims.
2. Use BUSINESS_MODEL.md to connect value, distribution, and revenue.
3. Use GTM.md and PRICING.md to design the next market test.

## Common Workflows

- Clarify an ICP and buying trigger.
- Draft a lean business model.
- Plan GTM experiments and pricing interviews.

## Quality Checks

- Assumptions are labeled and testable.
- Metrics are tied to learning or revenue, not vanity.
- Recommendations fit a small team with limited time.

## Useful Prompts

### Prompt 1

Challenge this startup idea and list the riskiest assumptions.

### Prompt 2

Create a 2-week GTM experiment plan for this ICP.

### Prompt 3

Review this pricing page for clarity, packaging, and willingness-to-pay signals.
