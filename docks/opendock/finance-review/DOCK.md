# Finance Review

개인·가구·소규모 팀의 월별 현금 흐름과 예산 차이를 민감정보 없이 검토하려는 사용자를 위한 dock입니다. 원천 데이터의 포함·제외 경계를 먼저 정하고 수입, 지출 카테고리, 반복 결제, 목표, 큰 지출, 이상 항목을 집계해 다음 달의 현실적인 조정안으로 연결합니다.

## 설치되는 내용

- `FINANCE_REVIEW_PLAYBOOK.md`: 집계 기준, 예산 차이, 이상 항목, 개인정보 처리 원칙
- `.opendock/templates/finance-review/RUN.md`: 기간·통화, source boundary, 대상 파일, 검증 기록 템플릿
- `.agents/skills/opendock-finance-review/`: 교육 목적 예산 검토 skill
- `.agents/workflows/opendock-finance-review/`: 데이터 경계부터 remediation까지의 workflow
- `.opendock/harness/opendock__finance-review/check.mjs`: 현재 run과 선언된 `finance/` 보고서만 검사하는 결정론적 gate

## 요청 예시

```text
6월 지출 요약으로 월간 예산 리뷰를 만들어줘. KRW 기준으로 카테고리별 예산·실제·차이를 계산하고 다음 달 조정안을 제안하되 계좌번호와 거래 원문은 결과에 넣지 마.
```

```text
최근 반복 결제를 찾아 유지·검토·취소 후보로 나눠줘. 확인되지 않은 청구를 사기로 단정하지 말고 다음 확인 단계와 불확실성을 적어줘.
```

```text
이번 달 큰 지출이 저축 목표에 어떤 영향을 줬는지 교육 목적의 예산 관점에서 설명해줘. 투자, 세금, 법률 결과를 보장하지 마.
```

## 작업 흐름

1. 검토 기간, 기준 통화, 포함·제외할 source data를 정합니다.
2. 원천 자료를 복제하지 않고 필요한 집계값만 사용합니다.
3. 수입, 지출, 반복 결제, 목표, 예산 차이, 큰 지출, 이상 항목을 사실·가정·조정안으로 분리합니다.
4. 공식 분류·정책 근거의 URL과 조회일을 기록합니다.
5. 결과를 사용자 소유 `finance/` 아래에 저장하고 run manifest에 대상 파일을 선언합니다.
6. 하네스 실패를 수정한 뒤 사람 또는 Codex가 예산 적합성을 별도로 검토합니다.

## 안전과 한계

- 계좌·카드·라우팅 번호, credential, 원본 명세서, 전체 거래 memo를 저장하지 않습니다.
- 개인·주거·여행 상세는 카테고리와 월 단위로 집계하고 정확한 주소·미래 이동 계획을 제거합니다.
- 이상 항목은 검토 신호일 뿐 사기나 위법으로 단정하지 않습니다.
- 이 dock은 교육 목적의 budgeting 도구이며 개인화된 투자·세무·법률 자문이나 결과 보장을 제공하지 않습니다.
- 하네스는 정적 구조와 고위험 패턴을 검사하며 계산의 경제적 결과나 외부 모델 재현성을 보장하지 않습니다.
