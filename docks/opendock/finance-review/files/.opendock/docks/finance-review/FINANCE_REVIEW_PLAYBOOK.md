# Finance Review Playbook

## 1. 기간, 통화와 데이터 경계

명확한 시작일·종료일과 하나의 기준 통화를 사용합니다. 환율을 쓰면 출처, 조회일과 적용일을 가정으로 기록합니다. 원천 자료는 읽기 전용 입력이며 결과에는 필요한 기간·카테고리 집계만 남깁니다.

계좌·카드·routing 번호, credential, 전체 거래 memo, 영수증 원문과 불필요한 위치 정보는 제외합니다. source file을 결과 폴더로 복사하지 않습니다.

## 2. 수입과 지출

수입은 세후·세전 기준, 반복·일회성 여부를 구분합니다. 지출은 사용자 목적에 맞는 안정된 category로 집계하며 환불과 계좌 간 이체를 소비로 중복 계산하지 않습니다.

## 3. 반복 결제와 목표

반복 결제는 제공자, 주기, 최근 금액, 다음 확인일과 유지·검토·취소 후보 상태를 기록합니다. 실제 취소는 자동 실행하지 않습니다. 목표는 기간, 목표액, 현재 진행과 이번 달 기여액으로 측정합니다.

## 4. 예산 차이와 큰 지출

category마다 예산, 실제와 차이를 같은 부호 규칙과 통화로 표시합니다. 큰 지출 기준은 사전에 정하고 일회성 여부, 목표 영향과 반복 가능성을 기록합니다.

## 5. 확인할 이상 항목

중복처럼 보이는 결제, 평소보다 큰 금액과 새 merchant는 확인 신호입니다. 사기나 위법으로 단정하지 말고 근거, 대안 설명, 확인 담당자와 날짜를 기록합니다.

## 6. 사실, 가정과 조정안

- 사실: 입력 집계와 공식 출처가 직접 뒷받침하는 값
- 가정: 분류, 환율, 일회성 여부처럼 아직 확인되지 않은 전제
- 조정안: 다음 달 budget category 또는 행동 변경 제안

변경 가능한 분류·정책 근거에는 URL과 조회일을 남깁니다.

## 7. 선택 가능한 산출물

현재 요청에 필요한 항목만 선택합니다.

- 수입·지출과 예산 차이
- 반복 결제와 목표 진행
- 큰 지출과 확인할 이상 항목
- 사실·가정·조정안
- 불확실성과 개인정보 처리

## 8. 검토 기준

사용자가 검토를 요청하면 AI는 현재 결과물만 기간·통화 일관성, 중복 계산, 근거, 가정, 개인정보와 교육 목적 경계 기준으로 직접 확인합니다.

이 playbook은 budgeting 교육용이며 투자·세무·법률 자문이나 결과 보장을 제공하지 않습니다.
