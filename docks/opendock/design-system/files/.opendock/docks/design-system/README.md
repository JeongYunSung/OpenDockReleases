# Design System 사용 안내

제품의 토큰, 컴포넌트 상태와 접근성 규칙을 일관된 계약으로 정리합니다.

## 시작하기

1. `DESIGN_SYSTEM_PLAYBOOK.md`에서 필요한 입력과 설계 기준을 확인합니다.
2. agent에게 결과물을 직접 요청합니다: “현재 UI를 보고 색상 토큰과 버튼 상태 계약을 정리해줘.”
3. 여러 결정을 함께 정리해야 하면 `RUN.md`에서 필요한 section만 골라 사용합니다.

## 검토

평소에는 요청한 결과물을 바로 작성하거나 수정합니다. 사용자가 검토를 요청하면 현재 결과물만 `DESIGN_SYSTEM_PLAYBOOK.md` 기준으로 AI가 직접 검토하며, 과거 결과물이나 프로젝트 전체는 검사하지 않습니다.

사용자가 만든 결과물은 사용자 소유이며 Dock 업데이트나 제거 대상이 아닙니다.
