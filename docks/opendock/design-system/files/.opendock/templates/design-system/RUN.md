# Design System 작업 메모 (선택 템플릿)

현재 작업에 필요한 section만 남겨 사용하세요. 모든 section을 채울 필요는 없습니다.

## Principles

제품·사용자 맥락에 연결된 판정 가능한 원칙을 작성합니다.

## Naming

semantic naming, alias, versioning과 deprecation 규칙을 작성합니다.

## Color Tokens

canvas, surface, text, border, primary, focus와 semantic state 역할을 작성합니다.

## Type Tokens

display, heading, body, label, caption과 code 역할을 작성합니다.

## Spacing / Radius / Shadow

layout과 component 성격에 따른 역할을 작성합니다.

## Component States

component별 interaction state, feedback와 접근성 동작을 작성합니다.

## Accessibility

contrast, focus, 이름·역할·값, target size, zoom과 reduced motion 계약을 작성합니다.

## Responsive Behavior

container, breakpoint, reflow, overflow와 content priority를 작성합니다.

## Governance / Decision Log

owner, 결정, 근거, 대안, 영향과 변경 절차를 작성합니다.

## Adoption Plan

inventory, pilot, migration, 측정, deprecation과 rollback 조건을 작성합니다.

## Evidence

기존 제품 inventory, contrast 확인과 component 조사 근거를 작성합니다.
