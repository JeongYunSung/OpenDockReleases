# Error Investigator 사용 안내

오류 증거, 재현 조건, 원인 가설과 최소 수정안을 추적합니다.

## 시작하기

1. `ERROR_INVESTIGATION_PLAYBOOK.md`에서 조사와 안전 기준을 확인합니다.
2. agent에게 오류 조사와 수정을 직접 요청합니다: “이 오류 로그와 최근 변경을 근거로 원인을 조사해줘.”
3. 조사 근거를 따로 남길 필요가 있으면 `RUN.md`에서 필요한 section만 골라 사용합니다.

## 검토

평소에는 오류를 바로 조사하고 수정합니다. 사용자가 검토를 요청하면 현재 조사 결과만 `ERROR_INVESTIGATION_PLAYBOOK.md` 기준으로 AI가 직접 검토하며, 과거 조사나 프로젝트 전체는 검사하지 않습니다.

사용자가 만든 조사 결과는 사용자 소유이며 Dock 업데이트나 제거 대상이 아닙니다.
