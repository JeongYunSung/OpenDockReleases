# 여행 기록 플레이북

평소에는 별도 준비 없이 바로 기록을 만듭니다. 작업 메모가 필요하면 선택 템플릿에서 관련 섹션만 사용하고, 사용자가 검토를 요청하면 AI가 현재 결과물만 이 플레이북 기준으로 직접 검토합니다.

## 최소 원본 선택

이야기에 필요한 사진·메모만 선택하고 `photo-001`, `note-002` 같은 안전한 ID로 inventory를 만듭니다. 원본 절대 경로, 클라우드 공유 토큰, EXIF 좌표, 기기 일련번호와 불필요한 얼굴·연락처 정보는 기록하지 않습니다.

## 사실과 기억

날짜, 장소, 인물, 사건은 사진·메모·사용자 확인 중 어떤 근거인지 표시합니다. 확신할 수 없는 시각, 장소명, 대화와 감정의 원인은 `추정`, `기억상`, `확인 필요`로 남기며 매끄러운 서사를 위해 사실을 창작하지 않습니다.

## 인물과 동의

각 인물의 표시 이름, 실명·사진·직접 인용·위치 공개 범위와 동의 상태를 기록합니다. 동의가 없거나 철회되면 가명 사용, 인용 재서술, 얼굴·신원 단서 제외와 위치 모호화를 적용합니다. 민감한 건강·관계·직장 정보는 이야기 목적에 꼭 필요하고 명시적으로 동의한 경우만 사용합니다.

## 이야기 구조

시간선과 장소, 장면별 하이라이트, 사진 캡션, 감각적이되 확인 가능한 세부, 여행 전후의 생각이 어떻게 달라졌는지 회고를 연결합니다. 공유용과 개인 보관용이 모두 필요하면 독자와 공개 범위에 맞게 버전을 나눕니다.

## 공개 전 가림

상세 숙소·자택 위치, 실시간 동선, 티켓·예약 식별자, 얼굴·차량번호·연락처, 정밀 GPS와 EXIF를 확인합니다. 동의와 목적이 없는 데이터는 제거하고 원본은 변경하지 않습니다. 공개본과 개인 보관본의 범위를 구분합니다.

## 완료 기준

요청에 선택된 사진·메모의 사용 여부가 결과와 연결되어야 합니다. 현재 결과에 관련된 시간선, 캡션, 불확실성, 회고, 동의와 가림을 확인하며 불필요한 섹션이나 이야기 변형을 강제하지 않습니다.
