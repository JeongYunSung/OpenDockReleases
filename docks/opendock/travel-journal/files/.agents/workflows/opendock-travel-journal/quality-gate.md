# Travel Journal 품질 게이트

## 범위와 원본

템플릿을 run manifest로 복사하고 독자·공개 범위를 정합니다. 필요한 사진·메모만 안전한 ID로 inventory에 기록합니다.

## 사실과 동의

시간선과 장소를 근거에 연결하고 불확실성을 표시합니다. 인물별 가명, 공개 동의와 철회 시 제외·가림 계획을 확인합니다.

## 산출물

`travel-journal/` 아래에 하이라이트, 캡션, 회고, 개인정보 가림과 짧은·긴 이야기 변형을 작성해 manifest에 선언합니다.

## 검사와 승인

`node .opendock/harness/opendock__travel-journal/check.mjs <manifest-path>`를 통과할 때까지 수정합니다. 이후 Codex와 사람이 서사, 사실, 동의, 저작권과 공개 범위를 별도로 승인합니다.
