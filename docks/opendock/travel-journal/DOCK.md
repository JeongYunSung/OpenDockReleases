# Travel Journal

## 무엇을 알려주면 무엇이 나오나요

- 알려줄 것: 사진·메모의 안전한 이름, 여행 날짜, 장소 범위, 함께한 사람의 공개 동의
- 정리되는 것: `travel-journal/` 아래에 시간순 기록, 짧은 캡션, 긴 이야기와 공개용 가림 확인표
- 요청 예시: "교토 여행 사진 목록과 메모를 줄게. 동행자는 가명으로 쓰고 짧은 글과 긴 글을 travel-journal/kyoto.md로 정리해줘."

## 설치되는 안내

- `.opendock/docks/travel-journal/README.md`: 가장 쉬운 사용 순서
- `.opendock/docks/travel-journal/TRAVEL_JOURNAL_PLAYBOOK.md`: 자세한 작성 기준
- `.opendock/templates/travel-journal/RUN.md`: 필요할 때 일부만 쓰는 선택 양식

## 사용 순서

1. 필요한 정보를 받고 민감한 값은 먼저 뺍니다.
2. 확인된 날짜·장소와 기억이 불확실한 내용을 구분해 이야기와 캡션을 만듭니다.
3. 결과를 사용자 소유 `travel-journal/` 아래에 저장합니다.
4. 평소에는 별도 준비 없이 바로 작업하며, 양식은 필요한 섹션만 골라 사용합니다.

## 검토 요청

사용자가 검토를 요청하면 AI가 현재 결과물만 `TRAVEL_JOURNAL_PLAYBOOK.md` 기준으로 직접 검토합니다.

## 안전

- 동의 없는 실명, 얼굴 설명, 연락처와 신원 단서는 공개하지 않습니다.
- 정확한 GPS, 숙소·자택 위치와 원본 EXIF는 넣지 않습니다.
- 웹의 가격·규정·운영 정보는 출처와 확인 날짜를 남기고 중요한 결정 전에 다시 확인합니다.
- 사용자가 만든 `travel-journal/` 파일은 Dock을 업데이트하거나 제거해도 삭제하지 않습니다.
