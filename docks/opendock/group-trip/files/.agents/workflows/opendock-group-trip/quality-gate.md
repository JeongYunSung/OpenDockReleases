# Group Trip 품질 게이트

## 입력

템플릿을 run manifest로 복사하고 각 구성원의 선호, 제약, 예산과 접근성 요구를 같은 형식으로 받습니다. 가명을 사용하고 민감 특성을 추정하지 않습니다.

## 합의 설계

공통점과 갈등, 선택지별 tradeoff를 공개합니다. 결정 방식, 동점·기권·이의 제기 규칙, 비용 배분과 열린 투표를 작성합니다.

## 근거와 산출물

시간 민감 옵션에는 출처 URL과 접근일을 붙이고 사실, 가정, 추천을 분리합니다. 산출물은 `group-trip/` 아래에 작성해 manifest에 선언합니다.

## 검사와 승인

`node .opendock/harness/opendock__group-trip/check.mjs <manifest-path>`를 통과할 때까지 수정합니다. 이후 Codex 품질 검토와 모든 구성원의 공개 범위·결정 동의를 별도로 확인합니다.
