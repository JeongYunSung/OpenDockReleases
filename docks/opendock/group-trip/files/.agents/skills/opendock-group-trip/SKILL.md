# Group Trip

## 작업 순서

1. `.opendock/docks/group-trip/GROUP_TRIP_PLAYBOOK.md`를 읽습니다.
2. 구성원마다 같은 형식으로 직접 밝힌 선호, 제약, 예산과 접근성 요구를 확인합니다.
3. 공통점, 갈등, tradeoff와 보호 조건을 분리합니다.
4. 투표·점수·순번 등 공정한 방식과 동점·이의 제기 규칙을 제안합니다.
5. 일정 대안, 비용 배분과 열린 결정을 요청된 결과물로 바로 작성합니다.
6. 작업 메모가 유용하면 선택 템플릿에서 필요한 section만 사용합니다.
7. 사용자가 검토를 요청하면 현재 결과물만 playbook 기준으로 AI가 직접 검토합니다.

## 안전 경계

민감 특성 추정, 동의 없는 개인정보 공개, 실제 예약·결제, 비밀값 처리, 파괴 명령과 만족 보장은 금지합니다. 시점에 따라 바뀌는 정보는 최신 출처와 확인일을 남깁니다.
