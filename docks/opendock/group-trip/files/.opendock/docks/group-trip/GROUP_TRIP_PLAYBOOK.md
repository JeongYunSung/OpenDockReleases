# 공동 여행 플레이북

## 동일한 입력 형식

모든 구성원에게 선호, 피하고 싶은 항목, 필수 제약, 총예산·일예산, 접근성 지원, 양보 가능한 항목과 응답 기한을 같은 방식으로 묻습니다. 답하지 않은 내용을 추정하지 않습니다. 이름 대신 가명을 사용할 수 있고 민감정보는 직접 동의한 범위에서만 기록합니다.

## 합의 지도

- 공통점: 모두가 수용하거나 강하게 원하는 항목
- 갈등: 동시에 만족할 수 없거나 비용·시간이 충돌하는 항목
- tradeoff: 각 선택으로 누가 무엇을 얻고 포기하는지
- 보호 조건: 건강·접근성·안전처럼 단순 다수결로 침해하면 안 되는 조건

## 공정한 결정 방식

점수 투표, 순위 투표, 순번 선택 또는 무작위 추첨 중 목적에 맞는 방식을 사전에 합의합니다. 투표 항목, 마감, 동점 처리, 기권, 이의 제기와 재투표 조건을 공개합니다. 공정성은 결과 만족 보장이 아니라 규칙의 일관성과 검토 가능성입니다.

## 비용 배분

공동 고정비, 사용량 비례비, 개인 선택비와 접근성 지원 비용을 구분합니다. 누가 결제하고 언제 정산하는지, 환불·취소·노쇼와 환율 차이를 어떻게 처리하는지 합의합니다. 개인 결제 정보와 예약 식별자는 문서에 넣지 않습니다.

## 선택 가능한 산출물

현재 요청에 필요한 항목만 선택합니다.

- 구성원별 입력표와 공통점
- 갈등, tradeoff와 보호 조건
- 결정 방식과 열린 투표
- 일정·대안과 비용 배분
- 접근성 계획
- 출처, 사실·가정·추천과 개인정보 처리

## 검토 기준

사용자가 검토를 요청하면 AI는 현재 결과물만 입력 형식의 공정성, 민감한 제약 보호, 결정 규칙, 비용 배분, 대안과 개인정보 기준으로 직접 확인합니다. 최종 결정과 공개 범위는 구성원들이 직접 합의합니다.
