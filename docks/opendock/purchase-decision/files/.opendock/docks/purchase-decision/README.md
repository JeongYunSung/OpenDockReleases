# Purchase Decision 사용 안내

## 먼저 이렇게 요청하세요

- 알려줄 것: 사용 목적, 예산, 꼭 필요한 조건, 있으면 좋은 조건, 제외할 조건과 후보
- 받을 결과: `purchases/` 아래에 후보 비교, 탈락 이유, 총비용, 근거와 최종 확인 항목을 담은 결정표
- 예시: "출장용 14인치 노트북 후보 3개와 예산을 줄게. 무게와 배터리를 우선해 purchases/laptop.md로 비교해줘."

## 사용 순서

1. 필요한 정보를 알려주고 빼야 할 개인정보도 함께 말합니다.
2. 공식 사양과 가격을 우선 확인하고 사실, 가정, 추천을 나눠 적습니다.
3. 결과는 사용자 소유 `purchases/` 아래에 바로 저장합니다.
4. 필요하면 `.opendock/templates/purchase-decision/RUN.md`에서 관련 섹션만 골라 사용합니다.

## 검토

검토를 요청하면 AI가 현재 결과물만 `.opendock/docks/purchase-decision/PURCHASE_DECISION_PLAYBOOK.md` 기준으로 직접 확인합니다. 사용 사례와 기준, 공식 출처·조회일, 가중치와 점수 근거, 총소유비용, dealbreaker, 불확실성과 이해상충을 점검하고 필요한 내용을 수정합니다.

## 주의할 점

- 확인하지 않은 가격, 재고와 사양을 사실처럼 쓰지 않습니다.
- 결제 정보, 계정 번호, 상세 주소와 불필요한 여행 일정은 적지 않습니다.
- 바뀔 수 있는 정보는 링크와 확인 날짜를 남기고 실제 행동 전에 다시 확인합니다.
- `purchases/` 아래의 결과는 사용자 소유이며 Dock 업데이트나 제거 대상이 아닙니다.
