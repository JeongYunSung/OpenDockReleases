# Purchase Decision Quality Gate

1. 요청을 제품, 서비스, 구독, 교체 결정 중 하나로 분류합니다.
2. 최소한의 사용 사례와 예산·기간을 확인하고 불필요한 주거·여행·계정 정보는 제거합니다.
3. Must, Should, Won't, dealbreaker와 가중치를 확정합니다.
4. `RUN.md`에서 dock별 run manifest를 만들고 대상 파일을 선언합니다.
5. 공식 출처를 우선해 후보별 사실, URL, 조회일을 수집하고 가정을 분리합니다.
6. 후보를 같은 척도로 비교하고 100% 가중치, 총소유비용, 위험, 이해상충을 검토합니다.
7. 추천 이유와 결제 전 다음 검증 단계를 작성해 `purchases/` 아래에 저장합니다.
8. purchase-decision harness를 실행하고 rule id별로 수정·재실행합니다.
9. 사람 또는 Codex가 추천 적합성을 별도로 수용 검토합니다. 외부 모델 결과의 재현성을 주장하지 않습니다.
