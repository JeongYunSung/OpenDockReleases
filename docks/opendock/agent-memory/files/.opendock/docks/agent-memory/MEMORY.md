# 프로젝트 Memory 가이드

AI가 다음 작업에서 다시 사용할 가치가 있는 맥락만 간결하게 기록합니다.

## 기록 후보

- 프로젝트 구조와 주요 모듈
- 중요한 결정과 이유
- 반복되는 확인 방법과 결과 해석
- 외부 도구의 프로젝트 전용 경로
- 알려진 위험, 공백과 오래될 수 있는 정보

모든 항목을 채울 필요는 없습니다. 다음 작업의 시작 비용을 실제로 줄이는 항목만 선택합니다.

## 기록하지 말 것

- API key, access token, refresh token, password, private key
- 개인 식별정보와 private prompt 전문
- 다음 작업에 필요하지 않은 대화나 로그 원문
- 근거 없는 추측을 확정 사실로 표현한 내용

## 권장 기록 형식

```markdown
## YYYY-MM-DD 주제

- 사실:
- 추측:
- 근거:
- 다음 확인:
```

필요한 줄만 남겨 사용합니다.

## AI 검토 기준

사용자가 검토를 요청하면 AI는 현재 결과물에서 사실과 추측의 구분, 근거의 추적성, 최신성 표시, 다음 확인과 민감 정보 노출만 직접 확인합니다.
