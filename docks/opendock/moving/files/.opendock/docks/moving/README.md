# Moving 사용 안내

## 먼저 이렇게 요청하세요

- 알려줄 것: 이사일, 출발·도착 지역, 가족과 반려동물, 건물 조건, 가구 치수, 예산
- 받을 결과: `moving/` 아래에 날짜별 준비 일정, 견적 비교, 주소·서비스 변경, 반입 확인과 당일 비상 계획
- 예시: "9월 12일 이사 조건과 가구 치수를 줄게. D-30부터 이사 후까지 할 일을 moving/plan.md로 정리해줘."

## 사용 순서

1. 필요한 정보를 알려주고 빼야 할 개인정보도 함께 말합니다.
2. 날짜별 할 일과 견적·반입·공과금·주소 변경 근거를 함께 정리합니다.
3. 결과는 사용자 소유 `moving/` 아래에 바로 저장합니다.
4. 필요하면 `.opendock/templates/moving/RUN.md`에서 관련 섹션만 골라 사용합니다.

## 검토

검토를 요청하면 AI가 현재 결과물만 `.opendock/docks/moving/MOVING_PLAYBOOK.md` 기준으로 직접 확인합니다. 일정·담당자·선행 조건, 견적 비교, 치수와 반입, 지역 정보의 출처·조회일, 예산과 비상 계획을 점검하고 필요한 내용을 수정합니다.

## 주의할 점

- 상세 주소, 공동현관 번호, 신분증 번호와 결제 정보는 적지 않습니다.
- 예약, 결제, 계약과 주소 변경 제출은 사용자의 확인 없이 진행하지 않습니다.
- 바뀔 수 있는 정보는 링크와 확인 날짜를 남기고 실제 행동 전에 다시 확인합니다.
- `moving/` 아래의 결과는 사용자 소유이며 Dock 업데이트나 제거 대상이 아닙니다.
