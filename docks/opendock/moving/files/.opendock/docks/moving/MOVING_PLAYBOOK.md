# Moving Playbook

## 1. 입력 기준

먼저 다음 값을 확정하거나 명시적인 가정으로 기록합니다.

- 이사일과 열쇠 인수·반납일
- 출발지와 도착지의 도시·구, 주거 유형, 층, 엘리베이터·주차·작업 시간 제약
- 성인·아동·반려동물·접근성 요구와 당일 부재자
- 포장, 운송, 청소, 보관, 조립 중 사용자가 맡을 범위
- 전체 예산과 이미 확정된 비용

정확한 주소와 호수는 경로·예약에 실제로 필요해질 때까지 일반화합니다. 공동현관 번호, 열쇠 보관 위치, 신분증 번호, 계좌·카드 정보는 계획에 남기지 않습니다.

## 2. 일정 구조

| 시점 | 최소 결과 |
|---|---|
| D-30 | 범위 확정, 재고 초안, 업체 후보와 견적 요청, 건물 규정 확인 |
| D-14 | 업체 선택, 서비스 이전 신청, 처분·기부 예약, 포장 시작 |
| D-7 | 주소 변경 목록 점검, 냉장·위험물 처리, 치수와 반입 경로 재확인 |
| D-1 | 필수품 가방, 귀중품 인계표, 연락망, 결제 조건, 날씨·교통 확인 |
| 당일 | 계량기·상태 사진, 상차·하차 재고, 손상 기록, 열쇠 인수·반납 |
| 이사 후 | 서비스 작동, 주소 반영, 손상 claim, 보증금·잔금, 미완료 작업 종료 |

각 작업에는 기한, 담당자, 선행 조건, 완료 증거를 둡니다. 날짜가 지났다면 과거 단계로 되돌리지 말고 미완료 항목을 현재 단계의 우선순위로 옮깁니다.

## 3. 업체와 비용

견적은 업체명, 견적일, 포함·제외 범위, 차량·인력, 추가 비용 조건, 취소 조건, 세금, 총액을 같은 기준으로 비교합니다. 구두 약속은 사실이 아니라 확인 대기 가정으로 표시합니다. 최저가만으로 권고하지 말고 보험·보상 범위와 현장 조건을 함께 봅니다.

예산은 운송, 포장, 청소, 보관, 폐기, 설치, 행정, 식사·교통, 예비비로 분리하고 확정액과 추정액을 구분합니다.

## 4. 물품과 반입

- 재고에는 품목, 수량, 상태, 포장 번호, 목적 방, 담당자를 기록합니다.
- 대형 품목은 폭·깊이·높이와 분해 가능 여부를 기록합니다.
- 현관, 복도, 계단 회전부, 엘리베이터 문·내부, 목적 방 문 치수를 같은 단위로 비교합니다.
- 단순 폭 비교만으로 반입을 보장하지 않습니다. 회전 반경, 포장 두께, 손잡이·몰딩, 대각선, 작업 여유를 가정으로 명시합니다.

## 5. 생활 서비스와 주소

전기, 수도, 가스, 인터넷, 통신, 보험, 관리사무소, 주차, 우편, 금융, 고용·교육·의료 등 실제 적용 대상을 목록화합니다. 각 항목은 현재 계정 종료일, 새 계정 시작일, 담당자, 확인 번호의 보관 위치를 기록하되 계정 번호 자체는 마스킹합니다.

## 6. 처분, 귀중품, 비상 계획

처분·기부는 품목, 방법, 마감일, 운반 책임, 지역 규정 출처를 기록합니다. 귀중품·문서·약·반려동물 용품은 이사업체 상차 대상과 분리하고 인계 책임자를 지정합니다.

비상 계획에는 최소한 업체 지연·취소, 차량·엘리베이터 문제, 악천후, 반입 실패, 파손, 열쇠 지연의 trigger, 연락 순서, 대체 행동, 추가 예산 한도를 둡니다.

## 7. 근거와 완료

주차 허가, 엘리베이터 예약, 폐기물 배출, 주소 이전처럼 지역·날짜 의존성이 있는 항목은 공식 HTTPS URL과 조회일을 남깁니다. 사실, 가정, 권고를 분리하고 권고에는 영향을 준 비용·치수·제약을 연결합니다.

완료 체크리스트가 필요한 경우 체크박스로 만들고 문서 제출이나 현장 확인이 필요한 항목의 증거 위치를 적습니다. 현장에서 확인하지 못한 항목은 별도로 인계합니다.

## 8. 작업과 검토

평소에는 요청한 이사 계획이나 목록을 바로 작성하고, 선택 템플릿에서는 관련 섹션만 사용합니다. 사용자가 검토를 요청하면 AI가 현재 결과물만 이 playbook 기준으로 직접 확인하고 일정, 담당자, 비용, 치수, 지역 정보 근거와 비상 계획 문제를 수정합니다.
