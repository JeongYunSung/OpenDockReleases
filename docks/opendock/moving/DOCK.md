# Moving

## 무엇을 알려주면 무엇이 나오나요

- 알려줄 것: 이사일, 출발·도착 지역, 가족과 반려동물, 건물 조건, 가구 치수, 예산
- 정리되는 것: `moving/` 아래에 날짜별 준비 일정, 견적 비교, 주소·서비스 변경, 반입 확인과 당일 비상 계획
- 요청 예시: "9월 12일 이사 조건과 가구 치수를 줄게. D-30부터 이사 후까지 할 일을 moving/plan.md로 정리해줘."

## 설치되는 안내

- `.opendock/docks/moving/README.md`: 가장 쉬운 사용 순서
- `.opendock/docks/moving/MOVING_PLAYBOOK.md`: 자세한 작성 기준
- `.opendock/templates/moving/RUN.md`: 필요한 항목만 골라 쓰는 선택 템플릿
- `.agents/skills/opendock-moving/SKILL.md`: agent 작업 절차

## 사용 순서

1. 필요한 정보를 받고 민감한 값은 먼저 뺍니다.
2. 날짜별 할 일과 견적·반입·공과금·주소 변경 근거를 함께 정리합니다.
3. 템플릿은 필요한 섹션만 선택해 사용합니다.
4. 결과를 사용자 소유 `moving/` 아래에 바로 저장합니다.

## 검토

사용자가 검토를 요청하면 AI가 현재 결과물만 Moving Playbook 기준으로 직접 검토하고, 일정·담당자·비용·치수·근거·비상 계획 문제를 수정합니다.

## 안전

- 상세 주소, 공동현관 번호, 신분증 번호와 결제 정보는 적지 않습니다.
- 예약, 결제, 계약과 주소 변경 제출은 사용자의 확인 없이 진행하지 않습니다.
- 웹의 가격·규정·운영 정보는 출처와 확인 날짜를 남기고 중요한 결정 전에 다시 확인합니다.
- 사용자가 만든 `moving/` 파일은 Dock을 업데이트하거나 제거해도 삭제하지 않습니다.
