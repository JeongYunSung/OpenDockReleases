# Product Roast 사용 안내

제품 경험의 문제를 증거, 영향과 우선순위 중심으로 날카롭게 비평합니다.

## 시작하기

1. `.opendock/docks/product-roast/PRODUCT_ROAST_PLAYBOOK.md`에서 필요한 입력과 결과 기준을 확인합니다.
2. agent에게 “이 가입 흐름을 사용성 근거와 개선 우선순위 중심으로 비평해줘”처럼 결과를 바로 요청합니다.
3. 필요하면 `.opendock/templates/product-roast/RUN.md`에서 요청에 맞는 섹션만 골라 사용합니다.

## 검토

검토를 요청하면 AI가 현재 결과물만 Playbook 기준으로 직접 확인합니다. 관찰과 영향의 연결, severity 이유, 유지·변경 판단, 우선순위와 검증 신호, 사실·가정·추천 분리를 점검하고 필요한 내용을 수정합니다.

사용자가 만든 결과물은 사용자 소유이며, Dock이 설치한 파일이 아니라면 OpenDock 업데이트나 제거가 삭제하지 않습니다.
