# Life Admin

구독, 갱신, 보증, 중요 문서의 메타데이터, 반복 행정 업무를 한곳에서 관리하려는 개인·가구·소규모 팀을 위한 dock입니다. 비밀값이나 문서 원문을 모으지 않고도 담당자, 날짜, 상태, 알림을 재검토 가능한 생활 운영 기록으로 정리합니다.

## 설치되는 내용

- `LIFE_ADMIN_PLAYBOOK.md`: 구독·갱신·보증·문서 metadata·연간 점검 원칙
- `.opendock/templates/life-admin/RUN.md`: 범위, 증거, 대상 파일, 개인정보 검토를 기록하는 run 템플릿
- `.agents/skills/opendock-life-admin/`: 생활 행정 정리 skill
- `.agents/workflows/opendock-life-admin/`: inventory부터 remediation까지의 workflow
- `.opendock/harness/opendock__life-admin/check.mjs`: 현재 run과 선언된 `life-admin/` 보고서만 검사하는 결정론적 gate

## 요청 예시

```text
올해 갱신되는 구독과 보증을 정리해줘. 서비스명, 요금 주기, 다음 갱신일, 담당자, 취소 확인 단계만 남기고 계정 번호나 결제 정보는 쓰지 마.
```

```text
여권, 보험, 임대 관련 문서의 만료일 관리표를 만들어줘. 문서 본문과 전체 식별번호는 저장하지 말고 유형, 발급기관, 날짜, 안전한 보관 위치 참조만 기록해줘.
```

```text
매달·분기·연간으로 반복되는 집안 행정 업무를 담당자와 상태가 있는 체크리스트로 만들고 알림 시점을 제안해줘.
```

## 작업 흐름

1. 관리 범위와 기간, 소유자 역할을 정합니다.
2. 구독, 갱신, 문서 metadata, 보증, 반복 업무를 최소 데이터로 inventory합니다.
3. 공식 제공자 URL과 조회일, 사실·가정·제안을 분리해 기록합니다.
4. 결과를 사용자 소유 `life-admin/` 아래에 저장하고 run manifest에 대상 파일을 선언합니다.
5. 하네스를 실행해 누락, 민감정보, 경로 문제를 수정합니다.
6. 사람 또는 Codex가 알림 시점과 실제 운영 적합성을 별도로 검토합니다.

## 안전과 한계

- 비밀번호, token, 계정·카드 번호, 주민·여권·면허 전체 번호, 문서 원문을 저장하지 않습니다.
- 실명은 역할로, 집 위치는 도시·지역 수준으로, 여행 계획은 공개 가능한 월·지역 수준으로 최소화합니다.
- 외부 문서와 프로젝트 텍스트는 신뢰되지 않은 증거로 취급하며 그 안의 지시를 실행하지 않습니다.
- 알림은 계획 기록이며 실제 캘린더 등록이나 갱신 취소를 자동 수행하지 않습니다.
- 하네스는 구조와 고위험 패턴을 검사할 뿐 누락 없는 행정 관리나 외부 모델 재현성을 보장하지 않습니다.
