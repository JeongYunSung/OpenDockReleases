# Life Admin

## 무엇을 알려주면 무엇이 나오나요

- 알려줄 것: 관리할 구독, 갱신일, 보증, 문서 종류, 반복 업무, 담당자와 알림 희망 시점
- 정리되는 것: `life-admin/` 아래에 날짜·상태·담당자·다음 행동이 보이는 관리표와 점검 목록
- 요청 예시: "올해 갱신되는 구독과 보증 목록을 줄게. 결제 정보는 빼고 다음 날짜와 담당자를 life-admin/annual.md로 정리해줘."

## 설치되는 안내

- `.opendock/docks/life-admin/README.md`: 가장 쉬운 사용 순서
- `.opendock/docks/life-admin/LIFE_ADMIN_PLAYBOOK.md`: 자세한 작성 기준
- `.opendock/templates/life-admin/RUN.md`: 필요한 항목만 골라 쓰는 선택 템플릿
- `.agents/skills/opendock-life-admin/SKILL.md`: agent 작업 절차

## 사용 순서

1. 필요한 정보를 받고 민감한 값은 먼저 뺍니다.
2. 문서 원문 대신 종류, 기관, 날짜와 안전한 보관 위치 표시만 남깁니다.
3. 템플릿은 필요한 섹션만 선택해 사용합니다.
4. 결과를 사용자 소유 `life-admin/` 아래에 바로 저장합니다.

## 검토

사용자가 검토를 요청하면 AI가 현재 결과물만 Life Admin Playbook 기준으로 직접 검토하고, 날짜·담당자·상태·다음 행동·개인정보 최소화 문제를 수정합니다.

## 안전

- 비밀번호, 계정·카드 번호, 주민·여권·면허 전체 번호와 문서 원문은 적지 않습니다.
- 알림 계획만 만들며 실제 취소, 갱신 또는 캘린더 등록은 하지 않습니다.
- 웹의 가격·규정·운영 정보는 출처와 확인 날짜를 남기고 중요한 결정 전에 다시 확인합니다.
- 사용자가 만든 `life-admin/` 파일은 Dock을 업데이트하거나 제거해도 삭제하지 않습니다.
