# Life Admin Quality Gate

1. 기간과 생활 행정 범위를 정하고 필요한 개인정보를 최소화합니다.
2. 구독, 갱신, 문서 metadata, 보증, 반복 업무를 inventory합니다.
3. 공식 URL과 조회일을 확인하고 사실, 가정, 제안을 구분합니다.
4. `RUN.md`에서 dock별 run manifest를 만들고 `life-admin/` target을 선언합니다.
5. 문서 본문·전체 식별번호를 제거하고 담당자·날짜·상태를 표준화합니다.
6. 알림과 연간 checklist를 작성하고 주거·여행·사람 정보를 안전한 수준으로 낮춥니다.
7. life-admin harness를 실행하고 rule id별로 수정·재실행합니다.
8. 사람 또는 Codex가 실제 일정과 운영 적합성을 별도로 검토합니다. 외부 모델 재현성을 주장하지 않습니다.
