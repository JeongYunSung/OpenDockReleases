# Life Admin 사용 안내

## 먼저 이렇게 요청하세요

- 알려줄 것: 관리할 구독, 갱신일, 보증, 문서 종류, 반복 업무, 담당자와 알림 희망 시점
- 받을 결과: `life-admin/` 아래에 날짜·상태·담당자·다음 행동이 보이는 관리표와 점검 목록
- 예시: "올해 갱신되는 구독과 보증 목록을 줄게. 결제 정보는 빼고 다음 날짜와 담당자를 life-admin/annual.md로 정리해줘."

## 사용 순서

1. 필요한 정보를 알려주고 빼야 할 개인정보도 함께 말합니다.
2. 문서 원문 대신 종류, 기관, 날짜와 안전한 보관 위치 표시만 남깁니다.
3. 결과는 사용자 소유 `life-admin/` 아래에 바로 저장합니다.
4. 필요하면 `.opendock/templates/life-admin/RUN.md`에서 관련 섹션만 골라 사용합니다.

## 검토

검토를 요청하면 AI가 현재 결과물만 `.opendock/docks/life-admin/LIFE_ADMIN_PLAYBOOK.md` 기준으로 직접 확인합니다. 날짜·담당자·상태·다음 행동, 출처와 조회일, 사실·가정·제안 분리와 개인정보 최소화를 점검하고 필요한 내용을 수정합니다.

## 주의할 점

- 비밀번호, 계정·카드 번호, 주민·여권·면허 전체 번호와 문서 원문은 적지 않습니다.
- 알림 계획만 만들며 실제 취소, 갱신 또는 캘린더 등록은 하지 않습니다.
- 바뀔 수 있는 정보는 링크와 확인 날짜를 남기고 실제 행동 전에 다시 확인합니다.
- `life-admin/` 아래의 결과는 사용자 소유이며 Dock 업데이트나 제거 대상이 아닙니다.
