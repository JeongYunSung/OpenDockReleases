# Research Desk Guide

자료 수집, 출처 비교, 주장/근거 분리, 인용 정리, 신뢰도 판단, 리서치 노트를 만드는 조사 workspace.

평소에는 별도 준비 없이 바로 조사합니다. 사용자가 검토를 요청하면 AI가 현재 결과물만 이 가이드 기준으로 직접 검토합니다.

## 원칙

- 리서치 질문, 의사결정 맥락, 필요한 최신성, 제외 범위를 먼저 정합니다.
- 출처는 primary/secondary/community/opinion으로 구분합니다.
- 주장과 근거를 분리하고, 각 근거에는 출처, 날짜, 신뢰도, 반대 증거를 기록합니다.
- 불확실하거나 오래된 정보는 결론이 아니라 gap으로 표시합니다.
- 최종 답변은 recommendation, confidence, next research action을 포함합니다.
- 실시간/법률/의료/금융 판단은 최신 출처 확인과 전문가 검토 필요성을 표시합니다.

## 표준 작업 순서

1. 질문과 결정 맥락을 먼저 확정합니다.
2. 출처 유형과 최신성을 분리합니다.
3. 주장/근거/반대근거/gap을 표로 정리합니다.
4. 결론에는 confidence와 다음 확인 행동을 붙입니다.

## 선택 가능한 산출물 구조

요청과 의사결정에 필요한 section만 선택합니다. 아래 목록은 출발점이며 모든 section을 채울 필요는 없습니다.

- 리서치 질문
- 출처 목록
- 근거 표
- 신뢰도
- 빈틈
- 추천
- 다음 행동

## 품질 판단 기준

- 사용자가 바로 다음 행동을 알 수 있어야 합니다.
- 주장과 결정에는 근거 또는 source note가 있어야 합니다.
- blocker와 improvement를 섞지 않습니다.
- 담당자, 리스크, 후속 행동이 없는 문서는 handoff하지 않습니다.
- 검토 요청에서는 현재 결과물의 출처, 최신성, 반대 근거, 신뢰도와 다음 행동을 이 가이드에 따라 확인합니다.

## 예외 처리

예외는 다음 형식으로만 남깁니다.

```md
Approved Exception: <승인자 / 날짜 / 이유 / 만료 조건>
```

## 안전 경계

출처를 확인하지 못한 내용을 사실처럼 단정하지 않습니다.
