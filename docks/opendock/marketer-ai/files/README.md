# Marketer AI Workspace

Campaign briefs, content calendars, SEO briefs, and ad experiment planning.

## Who This Is For

Marketing teams, founders, content leads, and growth operators.

## What This Dock Sets Up

AI-assisted marketing starts from audience, offer, channel, evidence, and measurable experiment design.

## Installed Agent Context

- `AGENTS.md`: shared instructions for Codex and other agent runtimes.
- `CLAUDE.md`: Claude Code project memory and skill routing notes.
- `GEMINI.md`: Gemini-compatible workspace context.
- `.agents/skills/opendock-marketer-ai/SKILL.md`: OpenDock/OMA-style reusable skill instructions.
- `.codex/skills/opendock-marketer-ai/SKILL.md`: Codex skill entry for this workspace role.
- `.claude/skills/opendock-marketer-ai/SKILL.md`: Claude Code skill entry for this workspace role.
- `.cursor/rules/opendock-marketer-ai.mdc`: Cursor project rule for this dock.

## Start Here

1. Define audience, pain, offer, and conversion goal in CAMPAIGN_BRIEF.md.
2. Use CONTENT_CALENDAR.md to keep publishing consistent.
3. Use AD_EXPERIMENTS.md for testable hypotheses rather than random creative variants.

## Common Workflows

- Draft a campaign brief.
- Plan a month of content around one ICP.
- Design SEO or paid experiments with measurable outcomes.

## Quality Checks

- Claims are supportable and not misleading.
- Copy matches audience maturity and channel context.
- Experiments have a hypothesis, variant, metric, and decision rule.
