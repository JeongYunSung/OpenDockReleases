## Result Files

- `audits/ux/작성하세요.md`

결과 파일이 필요할 때만 경로를 적고, 아래 section 중 감사 범위에 관련된 것만 사용합니다.

## Scope

감사할 사용자, 흐름, 플랫폼, viewport, 시작·종료 상태와 제외 범위를 작성하세요.

## Product Context

제품 목적, 사용자 목표, 제약과 성공 조건을 작성하세요.

## Findings

각 finding의 ID, 관찰, 재현 조건, `Evidence:`, `Recommendation:` 연결을 작성하세요.

## Severity

각 finding의 심각도와 사용자 영향 근거를 작성하세요.

## Evidence

화면 위치, 재현 단계, 제공 자료 식별자와 관찰 사실을 작성하세요. 제공되지 않은 analytics는 만들지 마세요.

## Recommendation

변경 대상, 기대 동작, 검증 방법이 분명한 개선안을 작성하세요.

## Priority

영향, 확인된 빈도, 의존성을 바탕으로 실행 순서를 작성하세요.

## Accessibility

keyboard, focus, 이름·역할·값, 대비, 확대, 오류 인지 범위를 작성하세요.

## Responsive

작은 viewport, 긴 텍스트, 확대, 방향 전환, overflow 범위를 작성하세요.

## Copy

label, 오류 복구, 확인 문구, 용어 일관성 범위를 작성하세요.

## Data Boundaries

사용한 최소 데이터와 여행 일정, 집 주소, 정확한 위치, 실명, 연락처, 예약·계정 식별자 비식별화 내용을 작성하세요.

## Review Notes

사용자가 검토를 요청한 경우에만 확인한 항목, 수정 내용과 남은 한계를 작성하세요.

## Limitations

미검증 device, assistive technology, 사용자 연구와 불확실성을 작성하세요.
