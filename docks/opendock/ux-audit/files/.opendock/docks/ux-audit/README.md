# UX Audit 사용 안내
사용자 흐름을 관찰 근거, 심각도, 접근성과 개선안으로 진단합니다.
## 시작하기
1. `.opendock/docks/ux-audit/UX_AUDIT_PLAYBOOK.md`에서 필요한 입력과 결과 기준을 확인합니다.
2. agent에게 다음처럼 요청합니다: “이 결제 흐름을 모바일 접근성과 오류 회복 관점에서 점검해줘.”
3. 작업 메모가 필요하면 `.opendock/templates/ux-audit/RUN.md`에서 관련 섹션만 선택해 사용합니다.
## 작업과 검토
평소에는 별도 준비 없이 바로 작업합니다. 사용자가 검토를 요청하면 AI가 현재 결과물만 `UX_AUDIT_PLAYBOOK.md` 기준으로 직접 검토합니다.
사용자가 만든 결과물은 사용자 소유이며, Dock이 설치한 파일이 아니라면 OpenDock 업데이트나 제거가 삭제하지 않습니다.
