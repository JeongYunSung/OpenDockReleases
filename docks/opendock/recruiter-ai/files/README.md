# Recruiter AI Workspace

Job description, interview guide, scorecard, and candidate outreach workspace.

## Who This Is For

Recruiters, hiring managers, founders, and people teams.

## What This Dock Sets Up

Hiring materials become structured, criteria-based, and easier to review for fairness and role fit.

## Installed Agent Context

- `AGENTS.md`: shared instructions for Codex and other agent runtimes.
- `CLAUDE.md`: Claude Code project memory and skill routing notes.
- `GEMINI.md`: Gemini-compatible workspace context.
- `.agents/skills/opendock-recruiter-ai/SKILL.md`: OpenDock/OMA-style reusable skill instructions.
- `.codex/skills/opendock-recruiter-ai/SKILL.md`: Codex skill entry for this workspace role.
- `.claude/skills/opendock-recruiter-ai/SKILL.md`: Claude Code skill entry for this workspace role.
- `.cursor/rules/opendock-recruiter-ai.mdc`: Cursor project rule for this dock.

## Start Here

1. Clarify role outcomes before listing requirements.
2. Use SCORECARD.md to define evidence-based evaluation criteria.
3. Use INTERVIEW_GUIDE.md for consistent interview loops.

## Common Workflows

- Write a role description.
- Create an interview plan and scorecard.
- Draft candidate outreach and follow-up.

## Quality Checks

- Criteria are job-relevant and not biased proxies.
- Interview questions map to the scorecard.
- Outreach is honest, concise, and personalized.

## Useful Prompts

### Prompt 1

Rewrite this job description for clarity, fairness, and role outcomes.

### Prompt 2

Create interview questions mapped to this scorecard.

### Prompt 3

Draft a concise outreach message for this candidate profile.
