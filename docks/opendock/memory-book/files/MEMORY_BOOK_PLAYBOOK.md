# Memory Book Playbook

## 1. 기간, 테마, 독자

한 run에 시작일·종료일 또는 명확한 연도 범위와 하나의 중심 테마를 둡니다. 최종 독자와 공개 범위를 먼저 정해 동의와 가림 기준을 결정합니다.

## 2. Source inventory

각 source에 `[S1]` 같은 안정된 ID를 부여하고 유형, 대략적 날짜, 제공자 역할, 동의 상태, 사용 범위를 기록합니다. source media의 절대 로컬 경로, 계정 URL, credential은 기록하지 않습니다. 외부 처리가 필요하면 제공자에게 보내기 전에 명시적 승인을 받습니다.

## 3. 사실과 서사

timeline, highlights, captions의 사실 문장에는 source ID를 붙입니다. 서로 충돌하는 자료는 하나를 선택해 확정하지 말고 양쪽과 불확실성을 남깁니다. 인터뷰에 없는 발언, 사진으로 알 수 없는 감정, 자료가 없는 사건을 만들지 않습니다.

회고는 현재 작성자의 해석임을 분명히 하며 사실 section과 분리합니다.

## 4. 인물 동의

인물별로 표시 이름, 동의 상태, 허용 매체, 공유 범위를 기록합니다. 동의가 없거나 확인할 수 없으면 역할·가명·이니셜을 쓰고 private draft로 제한하거나 제외합니다. 미성년자와 민감한 사건은 더 좁은 공개 범위를 적용합니다.

## 5. 장소와 metadata

- 집, 학교, 직장, 여행 장소는 도시·지역·국가 수준으로 낮춥니다.
- 정확한 주소, 숙소명·객실, 미래 일정, 항공편, 반복 동선은 제외합니다.
- EXIF GPS, 위도·경도, 촬영기기 serial, face embedding 등 민감 metadata를 서사에 복사하지 않습니다.
- source 파일명도 개인 식별자가 포함되면 안전한 source ID로 대체합니다.

## 6. 캡션과 회고

캡션은 보이는 사실, 날짜·장소의 안전한 수준, source ID로 구성합니다. 사진만으로 관계나 감정을 추정하지 않습니다. 회고 질문은 변화, 감사, 배운 점을 다루되 다른 사람의 내면을 대신 말하지 않습니다.

## 7. Redaction과 출력 계획

제외·가림한 인물, 위치, 식별자, metadata를 기록합니다. 최종 plan에는 형식, 독자, 페이지/section 순서, caption 스타일, 공개 범위, 최종 동의 확인 단계를 포함합니다. PDF·인쇄물 생성은 별도 작업이며 이 dock의 Markdown harness는 binary 결과를 검증하지 않습니다.
