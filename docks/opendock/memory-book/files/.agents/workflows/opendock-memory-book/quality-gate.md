# Memory Book Quality Gate

1. 기간, 테마, 독자, 공유 범위를 확정합니다.
2. source별 ID, 유형, 날짜, 제공자 역할, 동의 상태를 inventory합니다.
3. `RUN.md`에서 dock별 run manifest를 만들고 `memories/` target을 선언합니다.
4. source ID를 근거로 timeline, highlights, captions를 만들고 reflection을 분리합니다.
5. 인물 동의와 공유 범위를 확인하고 미확인 인물은 가명·역할 또는 제외로 처리합니다.
6. 장소를 도시·지역 수준으로 낮추고 정확한 주소, 숙소, 상세 이동, EXIF/GPS와 식별자를 제거합니다.
7. redaction 기록, 최종 narrative/output plan, 검증·불확실성을 작성합니다.
8. memory-book harness를 실행하고 rule id별로 수정·재실행합니다.
9. 사람 또는 Codex가 사실성·동의를 별도로 검토하며 외부 모델 재현성을 주장하지 않습니다.
