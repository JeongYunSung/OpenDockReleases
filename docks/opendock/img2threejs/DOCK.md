# img2threejs

참고 이미지 속 물체나 캐릭터를 Three.js로 다시 만듭니다. 단순한 한 번의 코드 생성으로 끝내지 않고, 형태를 명세하고 단계별로 구현한 뒤 브라우저 렌더링을 원본과 비교하며 수정합니다.

결과는 사진에서 원본 3D 데이터를 추출한 파일이 아니라 primitive, procedural geometry와 material로 구성된 TypeScript 코드입니다. 회전, 애니메이션, 충돌 처리와 파괴 효과를 이어서 구현할 수 있도록 pivot, socket과 collider 구조도 함께 준비합니다.

## 이런 때 사용하세요

- 제품 사진을 웹에서 회전할 수 있는 Three.js 오브젝트로 만들 때
- 게임 소품이나 기계 부품을 코드로 재구성할 때
- 로고나 일러스트를 입체적인 웹 장면으로 바꿀 때
- 캐릭터 이미지를 실시간 렌더링용 스타일 모델로 만들 때
- 만들어진 모델을 원본 이미지와 비교하며 단계적으로 개선할 때

## AI에서 이렇게 사용하세요

AI에서 `$opendock-img2threejs` 스킬을 선택하고 이미지와 사용 목적을 알려주세요. 스킬 선택 기능이 없는 AI에서는 요청에 “img2threejs 방식으로”라고 적어도 됩니다.

### 요청 예시

> 이 제품 사진을 웹에서 마우스로 돌려볼 수 있는 Three.js 모델로 만들어줘. 실시간 성능을 우선하고 원본과 비교하면서 형태를 수정해줘.

> 이 로봇 이미지를 게임 소품으로 재구성해줘. 팔과 머리를 움직일 수 있게 pivot과 socket을 나누고, 각 단계의 렌더링을 검토해줘.

> 이 캐릭터 이미지를 스타일을 유지한 저폴리 Three.js 모델로 만들어줘. 사진 한 장으로 알 수 없는 부분은 추측이라고 표시하고 추가 이미지가 필요하면 먼저 알려줘.

## 작업 방식

1. 이미지가 3D 재구성에 적합한지 먼저 확인합니다.
2. 크기, 구조, 재질, 주요 디테일과 움직일 부분을 명세합니다.
3. blockout부터 구조, 형태, 재질, 조명과 interaction 순서로 구현합니다.
4. 각 단계에서 브라우저 렌더링과 원본 이미지를 나란히 비교합니다.
5. 중요한 특징이 맞지 않으면 명세나 코드를 수정한 뒤 다시 확인합니다.

OpenDock은 Python과 전용 Three.js preview 환경을 프로젝트 안에 준비합니다. 전역 패키지를 설치하지 않으며 다른 프로젝트의 실행 환경과 섞이지 않습니다.

## 사용 후 얻는 것

- 물체 구조와 재질을 기록한 `ObjectSculptSpec` JSON
- `THREE.Group`을 반환하는 TypeScript factory
- 애니메이션에 사용할 pivot, socket, collider와 destruction group
- 원본과 렌더링을 비교한 이미지와 단계별 검토 기록
- 실제 프로젝트에 옮길 수 있는 Three.js 소스 코드

## 검토가 필요할 때

요청에 `검토해줘`라고 덧붙이면 AI가 현재 모델의 명세, 생성 코드와 최종 렌더링만 다시 비교합니다. 관련 없는 프로젝트 파일을 전부 검사하거나 별도의 무거운 하네스를 실행하지 않습니다.

## 알아둘 점

- 사진 한 장에 보이지 않는 뒤쪽이나 가려진 형상은 정확히 복원할 수 없습니다.
- 사람의 얼굴이나 복잡한 캐릭터는 완전한 복제보다 스타일을 유지한 근사 결과에 적합합니다.
- 기본 출력은 `.glb`나 `.obj`가 아니라 Three.js TypeScript 코드입니다.
- 입력 이미지와 로고를 사용할 권리는 사용자가 확인해야 합니다.
- upstream [hoainho/img2threejs](https://github.com/hoainho/img2threejs)의 MIT 라이선스 코드와 품질 게이트를 프로젝트 로컬로 제공합니다.
