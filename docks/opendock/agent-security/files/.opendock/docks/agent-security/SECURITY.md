# AI 보안 검토 가이드

AI가 보안 민감 작업을 수행하거나 검토할 때 현재 변경의 신뢰 경계, 위험과 대응을 근거 중심으로 다룹니다.

## 민감 영역

- Auth, permission, role, admin
- Token, credential, API key, secret
- Webhook, MCP, registry, deploy
- Payment, PII, audit log, data export
- 외부 네트워크와 공급망 의존성

현재 변경과 관련된 영역만 선택합니다.

## 검토 항목

- 변경 이유와 범위
- 자산, 행위자와 신뢰 경계
- 악용 경로, 가능성, 영향
- 테스트, 코드 또는 문서 근거
- 완화책과 잔여 위험
- Rollback 또는 disable 경로

모든 항목을 채울 필요는 없습니다. 위험을 판단하고 대응하는 데 필요한 항목만 기록합니다.

## 금지

- 실제 secret 값을 문서나 응답에 노출하기
- 승인 없는 배포, 이전, 권한 변경
- 위험 확인을 실제 공격이나 외부 침투로 확대하기
- 출처와 내용을 검토하지 않은 외부 스크립트 권장하기

## AI 검토 기준

사용자가 검토를 요청하면 AI는 현재 결과물에서 민감 영역, 신뢰 경계, 악용 가능성, 영향, 근거, 완화책, rollback과 잔여 위험만 직접 확인합니다.
