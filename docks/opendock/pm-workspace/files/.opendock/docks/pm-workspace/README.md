# PM Workspace 사용 안내

제품 문제, 요구사항, 우선순위, 지표와 의사결정을 한 작업 흐름으로 관리합니다.

## 시작하기

1. `.opendock/docks/pm-workspace/PM_WORKSPACE_PLAYBOOK.md`에서 필요한 입력과 결과 기준을 확인합니다.
2. agent에게 “이 기능 요청을 문제, 비목표, 요구사항과 성공 지표로 정리해줘”처럼 결과를 바로 요청합니다.
3. 필요하면 `.opendock/templates/pm-workspace/RUN.md`에서 요청에 맞는 섹션만 골라 사용합니다.

## 검토

검토를 요청하면 AI가 현재 결과물만 PM Workspace Playbook 기준으로 직접 확인합니다. 사실·가정·미해결 질문 분리, 문제와 사용자, 요구사항, acceptance criteria, metric, risk, dependency와 decision 기록을 점검하고 필요한 내용을 수정합니다.

사용자가 만든 결과물은 사용자 소유이며, Dock이 설치한 파일이 아니라면 OpenDock 업데이트나 제거가 삭제하지 않습니다.
