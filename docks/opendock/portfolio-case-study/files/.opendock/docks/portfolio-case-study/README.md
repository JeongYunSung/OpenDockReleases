# Portfolio Case Study 사용 안내

프로젝트 경험을 근거 있는 문제, 기여, 과정과 결과의 사례 연구로 만듭니다.

## 시작하기

1. `.opendock/docks/portfolio-case-study/PORTFOLIO_CASE_STUDY_PLAYBOOK.md`에서 필요한 입력과 결과 기준을 확인합니다.
2. agent에게 “이 프로젝트 기록으로 포트폴리오 사례 연구 초안을 만들어줘”처럼 결과를 바로 요청합니다.
3. 필요하면 `.opendock/templates/portfolio-case-study/RUN.md`에서 요청에 맞는 섹션만 골라 사용합니다.

## 검토

검토를 요청하면 AI가 현재 결과물만 Playbook 기준으로 직접 확인합니다. 공개 범위, 본인과 팀의 기여 구분, claim과 source 연결, 수치의 측정 기간·귀속·한계, 개인정보와 metadata 가림을 점검하고 필요한 내용을 수정합니다.

사용자가 만든 결과물은 사용자 소유이며, Dock이 설치한 파일이 아니라면 OpenDock 업데이트나 제거가 삭제하지 않습니다.
