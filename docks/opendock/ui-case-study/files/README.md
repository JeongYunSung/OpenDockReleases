# UI Case Study Workspace

Portfolio case study templates for problem, process, decision rationale, and impact.

## Who This Is For

Product designers, UX designers, UI designers, and portfolio creators.

## What This Dock Sets Up

A case study that explains the problem, choices, evidence, and impact instead of only showing screenshots.

## Installed Agent Context

- `AGENTS.md`: shared instructions for Codex and other agent runtimes.
- `CLAUDE.md`: Claude Code project memory and skill routing notes.
- `GEMINI.md`: Gemini-compatible workspace context.
- `.agents/skills/opendock-ui-case-study/SKILL.md`: OpenDock/OMA-style reusable skill instructions.
- `.codex/skills/opendock-ui-case-study/SKILL.md`: Codex skill entry for this workspace role.
- `.claude/skills/opendock-ui-case-study/SKILL.md`: Claude Code skill entry for this workspace role.
- `.cursor/rules/opendock-ui-case-study.mdc`: Cursor project rule for this dock.

## Start Here

1. Fill in PROBLEM_DEFINITION.md before writing the narrative.
2. Use CASE_STUDY_TEMPLATE.md to structure the story.
3. Use IMPACT_MEASUREMENT.md to avoid overstating results.

## Common Workflows

- Create a case study outline.
- Rewrite a case study for clarity and evidence.
- Prepare interview talking points from a portfolio project.

## Quality Checks

- The case study names the user problem and constraints.
- Design decisions include rationale and alternatives.
- Impact is measured, qualified, or clearly marked as unavailable.
