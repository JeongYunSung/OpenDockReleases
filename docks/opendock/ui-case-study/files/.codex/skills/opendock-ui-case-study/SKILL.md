# UI Case Study Skill

## Purpose

A case study that explains the problem, choices, evidence, and impact instead of only showing screenshots.

## Use When

- Create a case study outline.
- Rewrite a case study for clarity and evidence.
- Prepare interview talking points from a portfolio project.

## Procedure

1. Restate the requested outcome in one sentence.
2. Read the relevant project files and the dock reference files listed below.
3. Identify assumptions, constraints, risks, and missing information.
4. Produce the smallest useful artifact: plan, review, template, implementation checklist, or finished content.
5. End with validation, review notes, and next action.

## Required References

- PROBLEM_DEFINITION.md
- CASE_STUDY_TEMPLATE.md
- IMPACT_MEASUREMENT.md
- PROMPTS.md

## Quality Checks

- The case study names the user problem and constraints.
- Design decisions include rationale and alternatives.
- Impact is measured, qualified, or clearly marked as unavailable.

## Prompt Starters

- Review this case study for credibility, clarity, and hiring-manager scanability.
- Turn these project notes into a case study outline.
- Rewrite this impact section without exaggerating results.
